## Code Review - PR 155: Add public portfolio approval detection

### Overview

This PR adds perceptual-hash (pHash/dHash) based detection of approved photos by comparing local files against CDN thumbnails scraped from bank public portfolio pages. The adapter pattern, pipeline separation, and hash-based matching approach are solid foundations. Below are issues to address before merge.

---

### Critical Issues

**1. Bug in `decision.py` - `or 999` falsy trap**

Line: `combined = evidence.phash_distance + (evidence.dhash_distance or 999)`

`0 or 999` evaluates to `999` in Python, not `0`. When `dhash_distance` is exactly `0` (identical images), this silently treats it as distance 999 and produces `NOT_FOUND`. Fix:

```python
combined = evidence.phash_distance + (
    evidence.dhash_distance if evidence.dhash_distance is not None else 999
)
```

Also: `999` is a magic number - define it as a named constant (e.g. `DHASH_MISSING_PENALTY = 999` in `constants.py`).

**2. Direct file I/O bypassing `shared/file_operations.py`** - violates CLAUDE.md

Multiple new files write directly instead of using the centralized file operations layer:
- `preview_downloader.py`: uses `open(cache_path)` and `os.makedirs()` directly
- `save_bank_session.py`: uses `open(cookies_path)` and `json.dump()` directly

These must use functions from `shared/file_operations.py` per the project rules.

**3. Redundant `import os` inside function body - `pipeline.py`**

`import os` appears inside `build_photo_record()` but `os` is already imported at module level. Remove the inner import.

---

### Architecture Issues

**4. Hardcoded `bank_name == "Pond5"` special-case in `pipeline.py`**

The `run_detection()` loop contains an explicit `if bank_name == "Pond5":` block that builds a portfolio index. This breaks the adapter abstraction. Pre-loading logic should be delegated to the adapter (e.g. `adapter.prepare(filtered_data, headless=headless)`).

**5. Cross-module path coupling**

Both `save_bank_session.py` and `pipeline.py` construct paths into the sibling `markphotomediaapprovalstatus/` module. That `public_portfolios.json` file is in `.gitignore`, so this path silently fails on a clean checkout. The committed `public_portfolios.defaults.json` should be the canonical source and live entirely within `markphotomediaapprovalstatusauto/`.

**6. Most bank adapters are empty stubs**

14 of 15 adapters (AdobeStock, Alamy, DepositPhotos, Dreamstime, etc.) return `[]` with TODO comments. Merging non-functional stubs creates false confidence those banks are supported. Options:
- Limit this PR to Pond5 only and add others in follow-up PRs, or
- Raise `NotImplementedError` in stubs so callers get a clear signal rather than silent empty results.

Add these to `.gitignore`.

**8. Inconsistent step numbering in `validate_pond5.py`**

Inline step comments jump: 3 -> 4 -> 9 -> 10 -> 7 (write files). Steps 5-8 are missing. Renumber or remove.

**9. Trailing blank lines in `remove_already_sorted_out.py`**

Three consecutive blank lines added before a log statement - inconsistent with Black formatting.

---

### Security

**10. User-Agent string hardcoded in `http_client.py`**

The Chrome impersonation header is a reasonable anti-bot bypass, but extract it to `constants.py` and add a comment explaining its purpose.

**11. Persistent Chromium profiles not fully covered by `.gitignore`**

`.gitignore` adds `cookies/` as a directory entry. Verify this covers the `profile_{Bank}/` subdirectories created by `save_bank_session.py`, which contain real session state (localStorage, IndexedDB). Add `cookies/**/` explicitly if needed.

---

### Test Coverage

**12. No test for `dhash_distance=0` combined path**

The `or 999` bug (issue 1) is not caught by existing tests. Add a `test_decision__scenarios` case where `dhash_distance=0` and `combined_threshold` is set, asserting the outcome is `FOUND`.

**13. `constants-test.py` not present**

CLAUDE.md requires test-only configuration via `constants-test.py`. New test files do not reference it.
